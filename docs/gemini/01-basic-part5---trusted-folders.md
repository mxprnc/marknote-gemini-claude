# 신뢰할 수 있는 폴더 (Trusted Folders) 가이드

<br/>

## 출처
- Trusted Folders guide : https://geminicli.com/docs/cli/trusted-folders/

<br/>

## 1. 신뢰할 수 있는 폴더란 무엇인가?
'신뢰할 수 있는 폴더(Trusted Folders)' 기능은 Gemini CLI가 특정 프로젝트의 강력한 기능들(자동 코드 실행, MCP 서버 연결 등)을 마음껏 사용할 수 있도록 허용할지 결정하는 보안 방어선입니다. 이를 통해 인터넷에서 다운로드한 신뢰할 수 없는 프로젝트가 사용자의 시스템 설정을 마음대로 바꾸거나 악성 명령어를 실행하는 것을 방지합니다.

<br/>

## 2. 기능 활성화 방법 (Pre-requisite)
이 기능은 사용자의 선택권을 위해 기본적으로 **비활성화**되어 있습니다. 기능을 사용하려면 전역 `settings.json` 파일에 아래 설정을 먼저 추가해야 합니다.

```json
{
  "security": {
    "folderTrust": {
      "enabled": true
    }
  }
}
```

<br/>

## 3. 작동 방식: 신뢰 대화상자 (The Trust Dialog)
기능 활성화 후 새로운 폴더에서 처음 Gemini CLI를 실행하면 세 가지 선택지가 나타납니다.

- **폴더 신뢰 (Trust folder)**: 현재 프로젝트에 모든 권한을 부여합니다.
- **상위 폴더 신뢰 (Trust parent folder)**: 상위 디렉토리를 신뢰하여 그 안의 모든 하위 프로젝트를 자동으로 신뢰 대상으로 포함합니다. (예: `~/projects` 폴더 전체 신뢰)
- **신뢰하지 않음 (Don't trust)**: 폴더를 제한된 상태로 유지합니다. CLI는 즉시 **'제한된 모드(Safe Mode)'**로 전환됩니다.

사용자의 선택은 `~/.gemini/trustedFolders.json`에 영구 저장됩니다.

<br/>

## 4. 탐색 단계와 보안 경고 (Discovery Phase)
결정을 내리기 전, CLI는 폴더 내부를 미리 훑어(Scan) 다음 항목들을 사용자에게 보고합니다.
- **커스텀 명령어**: 프로젝트 전용 단축 명령어가 존재하는지 여부.
- **MCP 서버 전용 설정**: 외부 서버에 연결하려는 시도가 있는지 확인.
- **후크(Hooks) 및 스킬(Skills)**: 시스템 동작을 가로채는 스크립트 존재 여부.
- **설정 덮어쓰기**: 전역 설정을 무시하고 변경하려는 로컬 설정.

이 과정에서 **보안 위험**(예: 특정 도구의 무조건 자동 승인 등)이 감지되면 빨간색 경고 메시지가 함께 표시되어 사용자의 신중한 판단을 돕습니다.

<br/>

## 5. '제한된 모드(Safe Mode)'의 제약 사항
폴더를 신뢰하지 않을 경우 보안을 위해 다음 기능들이 엄격히 차단됩니다.
1. **워크스페이스 설정 무시**: 프로젝트 전용 `.gemini/settings.json`을 로드하지 않습니다.
2. **환경 변수 차단**: 폴더 내의 `.env` 파일을 무시합니다.
3. **확장 기능 관리 금지**: 확장 기능을 설치하거나 업데이트할 수 없습니다.
4. **도구 자동 승인 비활성화**: 전역 설정과 관계없이 모든 동작에 사용자 컨펌이 필요합니다.
5. **메모리 로드 중단**: 프로젝트 전용 컨텍스트(GEMINI.md 등)를 자동으로 읽지 않습니다.
6. **MCP 서버 연결 거부**: 정의된 모델 컨텍스트 프로토콜 서버에 연결하지 않습니다.

<br/>

## 6. 신뢰 설정 관리 및 변경
- **현재 권한 재설정**: CLI 실행 중 `/permissions` 명령어를 입력하면 다시 신뢰 대화상자를 호출하여 상태를 바꿀 수 있습니다.
- **전체 목록 관리**: `~/.gemini/trustedFolders.json` 파일을 직접 열어 지금까지 내린 모든 신뢰 결정을 한눈에 확인하고 삭제할 수 있습니다.

---
**고급 팁**: VS Code와 같은 IDE 통합 모드를 사용하는 경우, IDE 자체에서 폴더를 신뢰하고 있는지 여부가 가장 높은 우선순위로 적용됩니다.
