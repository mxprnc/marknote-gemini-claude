## 출처
- Context with GEMINI.md : https://geminicli.com/docs/cli/gemini-md/

<br/>

## 1. GEMINI.md란 무엇인가?
`GEMINI.md` 파일은 Gemini 모델에게 프로젝트 고유의 지침이나 코딩 스타일 가이드 등을 전달하는 핵심 도구입니다. 매번 답변을 요청할 때마다 동일한 지침(예: "함수형으로 짜줘")을 반복할 필요 없이, 이 파일에 한 번만 적어두면 AI가 이를 인지하고 최적화된 답변을 제공합니다.

<br/>

## 2. 세밀한 컨텍스트 계층 구조 (Hierarchy)
Gemini CLI는 여러 위치에서 이 파일을 찾아 하나로 합친 뒤 모델에 전달합니다. 로드 순서는 다음과 같습니다.

1.  **전역 컨텍스트 (Global Context)**: 
    - **위치**: `~/.gemini/GEMINI.md`
    - **범위**: 모든 프로젝트에 공통적으로 적용되는 나만의 기본 원칙입니다.
2.  **워크스페이스 컨텍스트 (Workspace Context)**:
    - **위치**: 작업 중인 프로젝트의 루트 및 상위 디렉토리.
    - **범위**: 특정 프로젝트 전체에 적용되는 설계 원칙이나 스타일 가이드입니다.
3.  **실시간 서비스 컨텍스트 (Just-In-Time Context)**:
    - **위치**: 도구가 특정 폴더에 접근할 때, 해당 폴더 내부를 스캔합니다.
    - **범위**: 특정 모듈이나 컴포넌트에만 필요한 매우 구체적인 지침을 모델이 필요할 때만 불러옵니다.

<br/>

## 3. `/memory` 명령어 활용법
로드된 지침들을 실시간으로 확인하고 관리할 수 있습니다.

- **`/memory show`**: 여러 위치에서 로드되어 하나로 합쳐진 최종 컨텍스트 내용을 보여줍니다. AI가 지금 무슨 규칙을 따르고 있는지 점검할 때 유용합니다.
- **`/memory reload`**: 파일 내용을 수정한 후 즉시 AI에게 반영하고 싶을 때 사용합니다.
- **`/memory add [내용]`**: 전역 `GEMINI.md` 파일 끝에 새로운 기억을 즉시 추가합니다.

<br/>

## 4. 지침 모듈화 및 커스텀 (Advanced)

### ① 파일 임포트 (`@파일명.md`)
방대한 양의 지침을 하나의 파일에 두지 않고, `@./docs/style-guide.md`와 같은 문법으로 여러 파일에서 내용을 불러와 조합할 수 있습니다.

### ② 기본 파일명 변경
`GEMINI.md`라는 이름이 마음에 들지 않는다면 `settings.json`에서 원하는 이름으로 바꿀 수 있습니다.
```json
{
  "context": {
    "fileName": ["CONTEXT.md", "GEMINI.md"]
  }
}
```

---
**실전 예시 (GEMINI.md)**
```markdown
## TypeScript 코딩 스타일
- 인덴트는 공백 2칸을 사용합니다.
- 모든 인터페이스는 `I` 접두사를 붙입니다 (예: `IUserService`).
- 엄격한 타입 체크를 선호하며 `any` 사용을 금지합니다.
```

보이지 않는 유령 파트너처럼 작동하는 `GEMINI.md`를 통해 AI의 지능을 프로젝트 환경에 최적화해 보세요!
