# Rewind (대화 및 코드 되돌리기) 가이드

<br/>

## 1. Rewind란 무엇인가?
`/rewind` 명령어는 대화의 특정 지점으로 돌아가거나, 해당 시점 이후 AI가 수행한 코드 수정 내용을 선택적으로 취소할 수 있는 강력한 복구 도구입니다. 복잡한 개발 과정에서 AI의 실수를 바로잡거나, 다른 접근 방식을 시도할 때 매우 유용합니다.

<br/>

## 2. 사용 방법
- **명령어 입력**: 프롬프트 창에 `/rewind`를 입력하고 엔터를 누릅니다.
- **단축키**: **`Esc` 키를 두 번** 연속으로 누르면 즉시 되돌리기 화면이 열립니다.

<br/>

## 3. 인터페이스 및 조작법
명령을 실행하면 이전 작업 내역이 목록 형태로 나타나는 대화형 화면이 표시됩니다.

1. **지점 선택**: 방향키(위/아래)를 사용해 목록을 탐색합니다. 최신 상호작용이 목록 하단에 위치합니다.
2. **미리보기**: 선택한 항목에서 어떤 사용자 요청이 있었고, 몇 개의 파일이 수정되었는지 미리 확인할 수 있습니다.
3. **지점 확정**: 복구하고 싶은 프롬프트 지점에서 엔터를 누릅니다.
4. **실행 옵션 선택**: 다음 세 가지 중 상황에 맞는 복구 방식을 선택합니다.
   - **대화 및 코드 변경 모두 되돌리기**: 대화 기록과 파일 수정 내용을 모두 선택한 시점 이전으로 완벽히 되돌립니다.
   - **대화만 되돌리기**: 대화 기록만 지우고, AI가 수정한 코드는 현재 상태 그대로 유지합니다.
   - **코드 변경만 되돌리기**: 대화 기록은 남겨둔 채 파일 수정 내용만 이전으로 복구합니다.
   - **취소 (`Esc`)**: 복구 작업을 취소하고 프롬프트로 돌아갑니다.

*주의: 선택한 지점 이후에 AI가 수행한 코드 변경이 없는 경우, 코드 복구 관련 옵션은 나타나지 않습니다.*

<br/>

## 4. 핵심 주의 사항 (Key Considerations)

- **파괴적 작업 (Destructive Action)**: 되돌리기를 실행하면 현재 세션의 기록이나 파일 내용이 영구적으로 소실될 수 있으므로 신중하게 결정해야 합니다.
- **모델의 기억 상실**: 대화 기록을 되돌리면 AI 모델은 해당 시점 이후의 모든 대화 내용을 잊게 됩니다. 코드만 되돌리고 대화를 유지한 경우, 모델이 현재 파일 상태를 오해할 수 있으므로 변경 사실을 알려주는 것이 좋습니다.
- **수동 편집은 제외**: AI 전용 에디팅 도구를 통해 수정된 내용만 되돌릴 수 있습니다. **사용자가 직접 에디터에서 수정한 내용**이나 쉘 도구(`!`)를 통해 발생한 변경 사항은 소급 적용되지 않습니다.
- **압축 세션 대응**: 대화가 길어져 압축(Compression)된 상태에서도 저장된 세션 데이터를 기반으로 정확하게 과거 시점을 재구성하여 작동합니다.
