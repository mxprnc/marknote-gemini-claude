# 모델 라우팅 (Model Routing) 가이드

<br/>

## 출처
- Model routing official guide : https://geminicli.com/docs/cli/model-routing/

<br/>

## 1. 모델 라우팅이란 무엇인가?
모델 라우팅은 사용자가 선택한 주 모델(Primary Model)이 서버 오류, 할당량 초과(Quota Error) 등의 이유로 응답하지 못할 때, 자동으로 다른 가용 모델로 전환하여 작업을 지속할 수 있게 해주는 복구 기능입니다. 이 기능은 기본적으로 활성화되어 작업의 높은 가용성을 보장합니다.

<br/>

## 2. 작동 원리
`ModelAvailabilityService`가 각 모델의 상태를 실시간 모니터링하며 정의된 정책에 따라 요청을 스마트하게 분배합니다.

1.  **장애 발생**: 현재 사용 중인 모델이 실패(예: 429 Too Many Requests 등)하면 CLI는 즉시 대체 프로세스를 시작합니다.
2.  **사용자 동의**: 기본적으로 모델 전환 전 사용자에게 확인을 요청합니다.
    - *특이 사항*: 프롬프트 완성, 분류 등 내부의 작은 유틸리티 호출 시에는 `gemini-2.5-flash-lite` -> `gemini-2.5-flash` -> `gemini-2.5-pro` 순으로 사용자에게 묻지 않고 자동으로(Silent Fallback) 전환됩니다.
3.  **전환 완료**: 사용자가 승인하거나 자동 정책이 적용되면, 해당 턴 혹은 전체 세션 동안 대체 모델이 요청을 대신 처리합니다.

<br/>

## 3. 로컬 모델 라우팅 (실험적 기능)
호스팅된 모델의 호출 비용을 절감하기 위해, 어떤 모델을 사용할지 결정하는 '판단 지능'을 로컬에서 실행 중인 **Gemma** 모델에게 맡길 수 있습니다.

- **장점**: 비용 절감 및 호스팅 모델과 동등한 수준의 라우팅 결정 품질 유지.
- **요건**: Gemma 모델이 Gemini API 규격을 따르는 로컬 엔드포인트(HTTP) 뒤에서 작동 중이어야 합니다.

<br/>

## 4. 모델 선택 우선순위 (Selection Precedence)
Gemini CLI가 최종적으로 어떤 모델을 사용할지 결정하는 우선순위는 다음과 같습니다. (높은 지점의 설정이 우선함)

1.  **`--model` 플래그**: 실행 시 터미널에 직접 입력한 모델이 항상 최우선입니다.
2.  **`GEMINI_MODEL` 환경 변수**: 플래그가 없을 경우 환경 변수 설정값을 따릅니다.
3.  **`settings.json` 내 `model.name`**: 설정 파일에 정의된 기본 모델을 사용합니다.
4.  **로컬 모델 라우터**: 활성화된 경우 로컬 Gemma 모델이 작업에 가장 적합한 모델을 골라줍니다.
5.  **기본값 (Default)**: 위 설정이 모두 없을 경우 `auto` 모델로 자동 지정됩니다.
