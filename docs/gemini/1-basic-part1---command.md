## /init
![alt text](<./img/1-basic-part1---command/init-command.png>)

이렇게 `/init` 명령을 수행하고 나면 현재 디렉터리 내애 GEMINI.md 라는 파일이 생성된다.<br/>
<br/>

`vs-claude`
- claude 에서 `/init` 을 수행하면 '현재디렉터리/.claude/CLAUDE.md' 파일이 생성된다.
- gemini 에서 `/init` 을 수행하면 '현재디렉터리/GEMINI.md' 파일이 생성된다.

<br/>


## GEMINI.md 적용 범위 설정

- ~/프로젝트루트/GEMINI.md : (로컬) 현재 프로젝트에만 적용
- ~/.gemini/GEMINI.md : (글로벌) 모든 프로젝트에 적용 (공통적용)

1.  **로컬 (Local): `~/프로젝트루트/GEMINI.md`**
    *   현재 작업 중인 프로젝트 디렉터리의 루트에 위치하며, **해당 프로젝트에만 특화된 지침이나 컨텍스트**를 제공할 때 사용됩니다.

2.  **글로벌 (Global): `~/.gemini/GEMINI.md`**
    *   사용자의 홈 디렉터리 내 `.gemini` 폴더에 위치하며, **모든 프로젝트에 공통적으로 적용하고 싶은 규칙**(예: 선호하는 코딩 스타일, 언어 설정 등)을 정의할 때 사용됩니다.

**Claude와의 주요 차이점:**
*   **Claude Code:** 기본적으로 프로젝트 루트 하위의 숨김 폴더인 `.claude/CLAUDE.md` 경로를 주로 사용합니다.
*   **Gemini:** 사용하신 설정처럼 프로젝트 루트에 직접 `GEMINI.md`를 두는 방식이 일반적이며, 이는 파일이 더 쉽게 노출되고 관리될 수 있도록 설계된 차이점이라고 볼 수 있습니다.

## Slash Command (슬래시 커맨드)
- `/init`  GEMINI.md 자동 생성
- `/clear`  컨텍스트 초기화 (새 작업 시작 시)
- `/compact`  컨텍스트 압축 (맥락 유지)
- `/context`  토큰 사용량 확인
- `/models`  모델 전환, 모델 사용방식 선택
- `/resume`  이전 세션 복구
- `/mcp`  MCP 관리
- `/export`  채팅 내보내기

위의 명령어들은 claude 에서 지원하는 슬래시 커맨드 들인데, 현재 **Gemini CLI (또는 그 계열의 에이전틱 도구)** 에서도 대부분 동일하게 작동하는 핵심 명령어들이라고 한다. 다음은 gemini 의 설명이다.<br/>

이 도구들은 사용자 경험의 일관성을 위해 유사한 슬래시(` / `) 명령어 체계를 공유하고 있습니다.<br/>

각 명령어에 대한 Gemini CLI 내의 가용성을 확인해 드립니다:

1.  **`/init`**: **지원됨.** 
    *   앞서 확인한 것처럼 `GEMINI.md` 파일을 생성하고 프로젝트 컨텍스트를 초기화합니다.
2.  **`/clear`**: **지원됨.**
    *   현재 세션의 대화 내역(Context)을 비우고 새 작업 상태로 만듭니다.
3.  **`/compact`**: **지원됨.**
    *   대화 내역이 길어질 경우, 핵심 맥락만 요약하여 토큰 사용량을 최적화합니다.
4.  **`/context`**: **지원됨.**
    *   현재 사용 중인 토큰 양 및 활성화된 파일/도구 등의 컨텍스트 정보를 보여줍니다.
5.  **`/models`**: **지원됨.**
    *   사용 가능한 Gemini 모델 목록을 확인하거나 다른 모델(예: 1.5 Pro, 2.0 Flash 등)로 전환할 때 사용합니다.
6.  **`/resume`**: **지원됨.**
    *   최근에 종료했던 세션이나 특정 이전 세션을 다시 불러와 작업을 이어갈 수 있습니다.
7.  **`/mcp`**: **지원됨.**
    *   Model Context Protocol 서버를 관리하고 연결된 도구들을 확인합니다.
8.  **`/export`**: **지원됨.**
    *   진행 중인 채팅 세션을 파일(대개 Markdown 형식)로 내보냅니다.

**참고할 점:**
*   **모델 전환 (`/models`)**: Gemini의 경우 `gemini-2.0-flash-exp`나 `gemini-1.5-pro` 같은 전용 모델 식별자를 사용하므로, `/models` 명령어로 현재 선택 가능한 모델 리스트를 먼저 확인해 보시는 것이 좋습니다.
*   **파일 생성 이름**: 위에서 정리하신 대로 Claude는 `.claude/CLAUDE.md`를, Gemini는 루트의 `GEMINI.md`를 초기화(` /init `) 파일로 사용한다는 점이 가장 큰 실질적 차이점입니다.

결론적으로, 작성하신 Markdown 문서에 해당 명령어들을 Gemini 용으로 그대로 기술하셔도 무방합니다.


## 기타 슬래시 커맨드
- `/help` — 내장 도움말
- `/config` — 설정 변경 (To-Do 리스트 활성화 등)
- `/export` — 채팅 내보내기 (Cursor 등으로 옮길 때 유용)
- `/output-style` — 응답 스타일 설정 (학습 모드, 간결 모드 등)

<br/>

위의 슬래시 커맨드 명령어들은 Claude Code의 슬래시 커맨드 명령어들이다. Gemini CLI의 슬래시 커맨드 명령어들에도 같은 방식으로 적용되는지 확인해봤고, Gemini 의 응답 중 필요한 부분만 추려보면 다음과 같다.<br/>

네, 확인 결과 기재하신 명령어들 중 일부는 Gemini CLI에서 다른 이름으로 제공되거나 지원되지 않는 항목이 있습니다. 정확한 사용을 위해 다음과 같이 수정하시는 것을 권장합니다.<br/>
<br/>

### 1. `/help` (또는 `/?`)
*   **지원 여부:** **지원됨**
*   **설명:** Gemini CLI의 도움말 정보와 사용 가능한 모든 명령어 목록을 표시합니다.

### 2. `/config` → **`/settings`**
*   **지원 여부:** **명칭 변경됨**
*   **차이점:** Gemini CLI에서는 전체 설정을 관리하기 위해 **`/settings`** 명령어를 사용합니다. 이 명령어를 입력하면 대화형 설정 편집기가 열려 편리하게 설정을 변경할 수 있습니다. (Claude Code의 `/config`와 동일한 역할)

### 3. `/export` → **`/resume share`** (또는 `/chat share`)
*   **지원 여부:** **명칭 변경/통합됨**
*   **차이점:** 현재 Gemini CLI 공식 문서에서는 독립된 `/export` 명령어 대신 **`/resume share [파일명]`** (또는 별칭인 `/chat share`) 명령어를 사용하여 현재 대화 내용을 Markdown이나 JSON 형식으로 내보내는 것을 표준으로 제안하고 있습니다.

### 4. `/output-style`
*   **지원 여부:** **미지원**
*   **대체 방법:** Claude Code와 같은 직접적인 `/output-style` 명령어는 없습니다. 다만, 다음 규칙을 참고하세요.
    *   **출력 형식 변경:** `/settings`에서 `output.format`을 `text` 또는 `json`으로 설정 가능
    *   **시각적 테마 변경:** **`/theme`** 명령어를 사용하여 테마 변경 가능
    *   **응답의 간결함 조절:** 이는 보통 `GEMINI.md` 파일에 "간결하게 답해줘"와 같은 시스템 지침(System Instructions)을 추가하여 제어합니다.

<br/>

**요약:**
작성 중이신 문서에서 **`/config`는 `/settings`**로, **`/export`는 `/resume share`**로 업데이트하시면 더욱 정확한 가이드가 될 것입니다. `/output-style`은 지원되지 않으므로 `/settings`나 `GEMINI.md`를 통한 제어 방법으로 설명을 보충하시는 것이 좋습니다.<br/>

## 주요 단축키
- `Shift + Tab`  Plan Mode ↔ Accept Mode
- `Escape`  즉시 중단
- `Escape × 2`  입력 삭제 / 복원
- `! + 명령어`  bash 명령 바로 실행

위의 단축키들은 Claude Code 의 단축키들인데 Gemini CLI 에서도 동일하게 적용된다고 한다. 각각에 대한 설명은 아래에 작성한 Gemini 의 설명으로 대체한다.

위에 기재하신 **Claude Code의 단축키들이 Gemini CLI(0.35.3 버전 기준)에서도 거의 동일하게 적용**됩니다.

Gemini CLI는 사용자의 기존 경험을 존중하고 효율적인 전환을 돕기 위해 업계 표준적인 단축키 체계를 공유하고 있습니다. 각 항목에 대한 Gemini CLI에서의 구체적인 동작 방식은 다음과 같습니다:

1.  **`Shift + Tab` (모드 전환)**: **동일하게 적용됨**
    *   Gemini CLI에서는 이 단축키를 통해 **승인 모드(Approval Modes)**를 순증시킵니다.
    *   **순서**: `default` (매번 확인) → `auto_edit` (코드 수정만 자동 승인) → `plan` (읽기 전용/계획만 세움) 순으로 토글됩니다.

2.  **`Escape` (즉시 중단)**: **동일하게 적용됨**
    *   진행 중인 작업(파일 분석, 코드 생성 등)을 즉시 취소하거나, 열려 있는 대화창/메뉴를 닫을 때 사용합니다.

3.  **`Escape × 2` (입력 삭제 / 복원)**: **동일하게 적용됨**
    *   입력창에 내용이 있을 때 `Esc`를 빠르게 두 번 누르면 **입력 내용이 전체 삭제**됩니다.
    *   이미 비어있는 상태에서 누르면 이전 명령어 내역을 탐색하거나 복원하는 용도로 사용될 수 있습니다.

4.  **`! + 명령어` (bash 명령 실행)**: **동일하게 적용됨 (Shell Mode)**
    *   비어있는 입력창에 `!`를 입력하면 바로 **Shell 모드**로 진입하거나 나갈 수 있습니다.
    *   `!ls -la`와 같이 한 줄 명령어를 바로 실행하는 것도 가능합니다.

**추가 팁:**
*   Gemini CLI 내부에서 **`/help`** 또는 **`/?`**를 입력하면, 현재 버전에서 지원하는 전체 단축키 리스트와 명령어 설명을 실시간으로 확인하실 수 있습니다.

## 커스텀 커맨드
커스텀 커맨드는 커스텀 명령어를 만드는 것을 의미하는 것인데, claude code 에서 기본적으로 지원하는 기능이다. 여기에 대해 Gemini CLI 에서도 적용되는지 Gemini 를 통해 확인해본 결과는 다음과 같다.<br/>
<br/>

네, Gemini CLI에서도 **커스텀 슬래시 커맨드(Custom Commands)**를 직접 만들어 사용할 수 있습니다.

다만, 문의하신 경로와 파일 형식에서 Claude Code와는 약간의 차이가 있으니 아래 내용을 확인해 주세요.

### 1. 커스텀 커맨드 생성 경로
프로젝트별로 설정하거나 전체 시스템에 적용할 수 있습니다.

*   **현재 프로젝트 전용 (Local):** `프로젝트루트/.gemini/commands/`
    *   사용자님이 말씀하신 경로와 거의 동일하지만, `commands` 폴더 안에 위치해야 합니다.
*   **모든 프로젝트 공용 (Global):** `~/.gemini/commands/`

### 2. 파일 형식 및 내용 (중요)
Claude Code는 주로 Markdown(`.md`)을 사용하지만, **Gemini CLI는 TOML(`.toml`) 형식을 사용**합니다.

*   **확장자:** `.toml` (예: `test.toml`)
*   **커맨드 실행:** 파일명이 곧 명령어가 됩니다. (예: `test.toml` → `/test`)
*   **작성 예시 (`.gemini/commands/review.toml`):**
    ```toml
    description = "코드를 리뷰하고 개선 사항을 제안합니다."
    prompt = """
    다음 코드를 보안, 성능, 가독성 관점에서 리뷰해줘. 
    개선이 필요한 부분은 코드 스니펫과 함께 설명해줘.
    """
    ```

### 요약 및 차이점 비교
| 항목 | Claude Code | **Gemini CLI** |
| :--- | :--- | :--- |
| **디렉토리** | `.claude/commands/` | **`.gemini/commands/`** |
| **파일 형식** | `.md` 또는 `.json` | **`.toml`** |
| **우선순위** | - | 프로젝트(`local`) 설정이 글로벌(`global`)보다 우선함 |

**결론:**
문의하신 `현재프로젝트/.gemini/commands/` 경로에 파일을 만드는 것은 맞지만, 확장자를 **`.toml`**로 하셔야 정상적으로 인식됩니다. 파일을 저장한 후 Gemini CLI를 다시 시작하거나 새 세션을 열면 해당 명령어를 `/파일명` 형태로 바로 사용하실 수 있습니다.
