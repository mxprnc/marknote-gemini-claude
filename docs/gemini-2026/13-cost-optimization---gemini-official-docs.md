## 1. 토큰 캐싱(Token Caching)이란 무엇인가?
Gemini CLI는 API를 호출할 때 이전의 대화 내용(Context)이나 시스템 지침(System Instructions)을 지능적으로 재사용하여 처리 비용을 절감하는 **자동 토큰 캐싱** 기능을 지원합니다. 동일한 컨텍스트를 매번 반복해서 처리할 필요가 없으므로, 후속 요청 시 소모되는 유료 토큰량이 비약적으로 줄어듭니다.

<br/>

## 2. 지원 대상 및 제약 사항 (Availability)
모든 인증 방식에서 캐싱이 작동하는 것은 아닙니다. 현재 사용 중인 계정의 종류를 확인하세요.

| 인증 방식 | 지원 여부 | 상세 내용 |
| :--- | :---: | :--- |
| **Gemini API Key** | **지원 (✅)** | 개인 개발자용 API 키 사용 시 자동 적용 |
| **Vertex AI** | **지원 (✅)** | GCP 프로젝트 및 위치 기반 인증 시 적용 |
| **OAuth (코드 어시스트)** | **미지원 (❌)** | 구글 계정 로그인 연동 시에는 API 사양상 현재 캐시 생성을 지원하지 않음 |

<br/>

## 3. 절약된 비용 확인 방법 (`/stats`)
현재 진행 중인 대화 세션에서 얼마나 많은 토큰을 절약했는지 실시간으로 확인하려면 다음 명령어를 입력하세요.

```bash
/stats
```

### 주요 통계 항목 관련 설명
- **Input Tokens**: 모델로 전송된 새로운 입력 정보.
- **Cached Tokens**: 이전에 전송된 데이터 중 재사용되어 **비용이 청구되지 않은** 토큰 양입니다. 이 숫자가 클수록 비용 효율이 높음을 의미합니다.
- **Total Token Count**: 캐시를 포함하여 해당 세션에서 소모된 전체 컨텍스트 크기입니다.

<br/>

## 4. 효율적인 사용 팁
- **컨텍스트 유지**: 한 대화 주제를 정밀하게 이어갈수록 캐싱 효과가 극대화되어 장기적으로 비용이 저렴해집니다.
- **인증 방식 선택**: 비용 최적화를 최우선으로 한다면 **Gemini API Key**를 발급받아 사용하는 것이 유리할 수 있습니다.
- **주기적 점검**: 세션 도중 `/stats`를 확인하여 현재 프롬프트의 비용 효율성을 상시 점검하세요.
