## gstack
- CEO, 개발리드, 디자이너, QA, 보안담당 등 23개의 skill 존재
- "Nobody wants it" 이 되는 바이브코딩 애플리케이션의 단점에 대해 기획점/제품의 필요함 등등 부터 파악해나가는 과정을 거침
- 새로 시작하는 서비스는 왜 실패하는가?
  - 수요검증, 마켓리서치, 경쟁분석 이 안되어 있다. 
  - gstack 은 이 과정 역시 포함된다.
- Headless Chromium 데몬을 내장하고 있다. 이 헤드리스 크로미엄 데몬 덕에 백그라운드에서 헤드리스 하게 테스트가능하며, 쿠키, 로그인세션, 탭이 헤드리스 브라우저 레벨에서 유지됨
- 게리 탄 
  - "혼자서 50일 동안 주간 평균 1만줄의 코드, 100개의 PR 을 프로덕션 품질로 유지했다."
  - proactive (능동적인) 제안 : "이게 왜 안돼지?" 하는 자연어를 입력하면 /investigate 를 수행하고, "배포할께" 라고 하면 /ship 이 알아서 수행된다. 매번 명령어를 기억하고 있지 않아도 된다.

<br/>

## superpowers 와 다른 점
superpowers
- "코드를 어떻게 쓸지" 에 집중
- 7단계 TDD 강제. (브레인스토밍, 계획, 테스트 먼저 작성, 구현, 리뷰)
- 테스트 없이 코드 금지 - 규율
gstack
- gstack 은 역할중심, superpower 는 과정 중심
- gstack 과 superpowers 는 경쟁개념이 아니라 공존가능, 보완하는 것

<br/>

## 🛠️ 전체 워크플로우 요약
GStack은 **`Think → Plan → Build → Review → Test → Ship`** 순서로 스프린트가 진행되도록 설계되어 있습니다.

1.  **`/office-hours`**: 아이디어 검증 (할까 말까?)
2.  **`/plan-design-review`**: 설계 및 디자인 리뷰 (어떻게 만들까?)
3.  **`/review` & `/qa`**: 코드 리뷰 및 실제 구동 테스트 (잘 만들었나?)
4.  **`/ship`**: **최종 배포 및 PR 생성 (이제 내보내자!)**

> **요약하자면:** `/ship`은 "이제 다 됐으니 배포해!"라고 명령했을 때, AI가 **동기화, 테스트, 스타일 점검, PR 생성**까지 실수 없이 한 번에 처리해주는 **자동 출고 시스템**이라고 이해하시면 됩니다.

## /office-hours
- ceo 가 뭐 어디 직원한테 뭐 물어보는 거라고 했는데 이거 다른 영상에서 설명함
- e.g. "경쟁자는 있나요?", "왜 기존 서비스가 아니라 이걸 만들어야 하나요?"
- 이 과정을 거치면 이미 절반은 걸러진 상태에서 개발을 시작할수 있게 된다.(쓸데없는 제품을 안만들게 됨)
- gstack 을 쓸때 "세컨드 오피니언 받을까요?"하는 질문 역시 나오는데, 이건 다른 AI 모델에 질문을 하게 되는 걸 의미한다.

<br/>

## /plan-design-review
- 디자인 계획 세우는 작업 역시 AI가 진행
- 실제 사이트를 들어가서 버튼 눌러보고 페이지 이동하고 입력도 해보고
- 디자인 리뷰 "너무 흔한 패턴입니다", UI/UX 분석 "CTA가 약합니다" 와 같은 AI의 리뷰들이 계속 생기는데, AI가 만든걸 AI가 비판하고, 고치고를 반복하는 과정

<br/>

## /design-consultation
- 디자인 기준을 하나의 문서로 정리해준다.
- DESIGN.md 를 보고 하나의 문서로 정리하고 디자인 기준점 확립
- 새 화면에 자동 적용
- design 에 consistency 유지 가능
- 이 외에도 /design-shotgun (디자인 3개 안을 보여주는 등의 작업), /design-html 이 있다.

## /review

## /ship 
- 메인브랜치와 싱크, 테스트 전체 실행, 커버리지 감사, PR 생성까지 배포를 위한 단계
- 테스트 프레임워크가 없는 프로젝트일 경우 /ship 이 먼저 테스트 환경을 만들어준다.
- shipa 쉬파 배포하자!!!

## /cso 
- 보안책임자 역할
- 보안관점. 보안관점에서 취약점들 (e.g. OWASP 취약점, 하드코딩된 시크릿, 의존성 CVE 분석, SQL Injection, XSS Attack, Auth Bypass, API Rate Limit)
- pr 머지 전에 보안 감사를 돌리면 좋다.
