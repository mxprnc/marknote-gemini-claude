# 프롬프트 엔지니어링 기초 패턴

프롬프트의 성능을 극대화하기 위한 일반적인 구조와 주요 기법들을 정리합니다.

## 1. 일반적인 프롬프트 구조
효과적인 프롬프트는 다음과 같은 요소들로 구성됩니다.

| 요소 | 설명 | 예시 |
| :--- | :--- | :--- |
| **Role (역할)** | LLM에 부여할 페르소나 정의 | "You are a debate teacher." |
| **Task (작업)** | 수행해야 할 구체적인 작업 명령 | "Please create 10 candidate questions those are opposite to my opinion." |
| **Context (맥락)** | 작업에 필요한 배경 정보 및 상황 설명 | "I'm preparing 5th graders debate competition..." |
| **Example (예시)** | 모델이 참고할 수 있는 입출력 샘플 (Few-shot) | "This is sample example questions that you can refer..." |
| **Constraint (제약 사항)** | 형식, 단어 선택, 길이 등에 대한 제한 | "Use 5th grader level of English words only." |

---

## 2. 구분자(Separator) 사용
프롬프트 내에서 명령, 페르소나, 컨텍스트 등을 분리할 때 `###`, `"""`, `---` 등의 구분자를 사용하면 LLM이 각 요소를 더욱 명확하게 이해할 수 있습니다.

*   **나쁜 예시:**
    ```text
    아래 문장을 요약해줘 {문장}
    ```
*   **좋은 예시:**
    ```text
    아래 문장을 요약해줘
    
    내용: ###
    {문장}
    ###
    ```

---

## 3. 주요 프롬프트 기법

### 1) 제로샷 (Zero-Shot) 프롬프트
*   **정의:** 예제(Shot) 없이 질문만 바로 던지는 방식입니다.
*   **특징:** LLM의 기본 학습 성능에만 의존하며, 정확도가 낮고 환각(Hallucination) 발생 가능성이 높습니다.

**예시:**
```text
Write a short summary about Elon Musk.
Answer:
```

### 2) N-샷 (N-Shot / Few-Shot) 프롬프트
*   **정의:** 원하는 응답 형식을 유도하기 위해 1개 이상의 예시를 제공하는 방식입니다.
*   **특징:** 모델이 답변의 형식이나 스타일을 학습하여 훨씬 정교한 결과물을 냅니다.

**예시:**
```text
다음 제공되는 영화 리뷰가 긍정적인지 부정적인지 아래 예시를 참고해서 판별하세요.

예시 1:
리뷰: "이 영화는 정말 지루했어요. 스토리는 엉성하고, 연출은 밋밋하고..."
감성: 부정적

예시 2:
리뷰: "이 영화는 기대 이상이었어요! 스토리 전개가 흥미진진했고..."
감성: 긍정적

리뷰: "이 영화는 정말 최고였어요! 스토리, 연출, 연기 모두 완벽했고..."
감성:
```

### 3) Chain-of-Thought (CoT) 프롬프트
*   **정의:** 모델에게 "단계별로 생각하라"고 지시하거나, 풀이 과정을 포함한 예시를 제공하여 논리적 추론을 유도하는 방식입니다.
*   **특징:** 산술 문제, 상식 추론 등 여러 단계의 사고가 필요한 복잡한 작업에 유용합니다.
*   **효과:** 중간 추론 단계를 안내함으로써 복잡한 추론의 정확도를 높이며, N-shot 기법과 결합하여 사용할 때 더욱 강력합니다.

**예시 (Few-shot CoT):**
```text
[Model Input]
Q: 로저는 테니스 공을 5개 가지고 있습니다. 그는 테니스 공 2캔을 더 샀습니다. 각 캔에는 3개의 공이 들어있습니다. 현재 몇 개의 공을 가지고 있습니까?
A: 로저는 5개로 시작했습니다. 3개입 2캔은 6개입니다. 5 + 6 = 11. 정답은 11입니다.

Q: 식당에 사과가 23개 있었습니다. 점심을 만드는 데 20개를 사용하고 6개를 더 샀다면, 현재 사과는 몇 개입니까?
A:

[Model Output]
식당에는 원래 23개의 사과가 있었습니다. 점심을 만드는 데 20개를 사용했습니다. 그래서 23 - 20 = 3개가 남았습니다. 여기에 사과 6개를 더 샀으므로, 3 + 6 = 9가 됩니다. 정답은 9입니다.
```

### 4) Zero-Shot CoT (Chain-of-Thought)
*   **핵심:** 프롬프트 끝에 **"Let's think step by step(단계별로 생각해 보자)"**라는 문구를 추가하는 것만으로도 모델의 추론 성능을 비약적으로 향상시킬 수 있습니다.
*   **특징:** 별도의 Few-shot 예제 없이도 모델이 스스로 문제를 분할하여 논리적으로 접근하게 유도합니다.

**예시:**
```text
[Model Input]
Q: 저글러는 16개의 공을 저글링할 수 있습니다. 공의 절반은 골프공이고, 골프공의 절반은 파란색입니다. 파란색 골프공은 몇 개입니까?
A: 단계별로 생각해 보자. (Let's think step by step.)

[Model Output]
전체 공은 16개입니다.
공의 절반이 골프공이므로, 골프공은 8개입니다(16 / 2 = 8).
골프공의 절반이 파란색이므로, 파란색 골프공은 4개입니다(8 / 2 = 4).
따라서 정답은 4입니다.
```

### 5) ReAct (Reason + Act)
* CoT : Reasoning
* ReAct : Reasoning + Action
* Reason - Action - Result
*   **개념:** 언어적 추론(Reasoning)과 외부 환경과의 상호작용인 행동(Action)을 결합한 패턴입니다.
*   **구조 (Thought-Action-Observation):**
    1.  **Thought:** 도구를 사용하기 전, 무엇을 해야 할지 모델이 스스로 추론합니다.
    2.  **Action:** 검색, 계산기 사용 등 외부 도구(Tool)를 활용한 구체적인 행동을 수행합니다.
    3.  **Observation:** 행동의 결과(검색 결과 등)를 확인하고 다시 다음 단계를 추론합니다.
*   **활용:** LLM 에이전트 패턴(Plan and Execute)의 핵심 기법으로 사용됩니다.

**예시:**
**상세 예시 (AI 기술 조사 루프):**
```text
Question: UC Berkeley에서 발표한 'LWM(Large World Model)'이 무엇인지 알아보고, OpenAI의 Sora와 비디오 생성 길이 면에서 어떤 차이가 있는지 비교해줘.

Thought 1: 먼저 'LWM(Large World Model)'이 정확히 무엇인지, 그리고 어떤 기술적 특징을 가졌는지 검색해야 합니다.
Action 1: Search["UC Berkeley Large World Model (LWM) features"]
Observation 1: LWM은 수백만 개의 토큰을 한 번에 처리할 수 있는 초장기 문맥(Long-context) 추론 모델입니다. 특히 수천 시간 분량의 비디오 데이터를 학습하며, 매우 긴 비디오 생성이 가능하다는 특징이 있습니다.

Thought 2: LWM의 비디오 생성 길이에 대한 구체적인 수치를 확인했습니다. 이제 비교 대상인 OpenAI 'Sora'의 최대 비디오 생성 길이를 검색하여 비교해야 합니다.
Action 2: Search["OpenAI Sora max video duration"]
Observation 2: OpenAI의 Sora는 현재 한 번에 최대 60초(1분) 길이의 비디오를 생성할 수 있는 모델로 발표되었습니다.

Thought 3: 두 모델의 비디오 생성 역량에 대한 데이터가 수집되었습니다. LWM은 100만 토큰 이상의 컨텍스트를 지원하여 매우 긴 시퀀스를 처리하는 반면, Sora는 고품질의 1분 내외 비디오에 집중하고 있습니다. 이 내용을 정리하여 답변합니다.

Final Answer: LWM은 수백만 토큰의 초장기 문맥을 처리하여 수십 분 이상의 긴 비디오 흐름을 이해하고 생성하는 데 강점이 있는 반면, Sora는 현재 최대 1분 길이의 고품질 비디오 생성에 특화되어 있습니다.
```

**구성 요소 설명:**
*   **Thought (추론):** 현재 상황을 분석하고 다음에 어떤 행동이 필요한지 모델이 스스로 판단합니다.
*   **Action (행동):** 외부 도구(검색, 계산기 등)를 실제로 호출하는 단계입니다.
*   **Observation (관찰):** 실행한 행동의 결과(검색 내용 등)를 받아보고, 이를 바탕으로 다시 추론에 들어갑니다.
*   **Final Answer:** 충분한 정보가 수집되면 사용자에게 최종 답변을 제공합니다.
